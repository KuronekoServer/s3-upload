S3 Upload API — ドキュメント

概要

S3互換ストレージへファイルをストリーミングアップロードするAPIです。
ファイルはディスクに書き出さず直接アップロードされます。大容量ファイルや同時リクエストに対応しています。

エンドポイント

POST /api/v1/upload

ファイル1件をS3の uploads/{filename} キーへアップロードします。

リクエスト

ヘッダー / フィールド 種別 値 必須
Content-Type Header multipart/form-data 必須
X-Auth-Key Header 認証キー (AUTH_ENABLED=true の場合) 条件付き必須
file Form field アップロードするファイル 必須

リクエスト例

curl (認証なし) 
curl -X POST http://localhost:8080/api/v1/upload \
  -F "file=@/path/to/video.mp4"
curl (AUTH_ENABLED=true) 
curl -X POST http://localhost:8080/api/v1/upload \
  -H "X-Auth-Key: your-secret-key" \
  -F "file=@/path/to/video.mp4"
Python (requests) 
# pip install requests
import requests

with open("/path/to/video.mp4", "rb") as f:
    resp = requests.post(
        "http://localhost:8080/api/v1/upload",
        headers={"X-Auth-Key": "your-secret-key"},  # AUTH_ENABLED=true の場合
        files={"file": f},
    )
print(resp.json())

環境変数

変数名 デフォルト 説明
S3_ACCESS_KEY — 必須  S3 アクセスキー
S3_SECRET_KEY — 必須  S3 シークレットキー
S3_BUCKET — 必須  対象バケット名
S3_ENDPOINT — 必須  S3互換ストレージのエンドポイント URL
S3_REGION us-east-1 リージョン
MAX_CONCURRENT_UPLOADS 10 同時アップロード上限数
UPLOAD_PART_SIZE_MB 32 パートサイズ (MB)。大きいほど高速だがメモリ使用量が増加
UPLOAD_CONCURRENCY 8 1ファイルあたりの並列パート数。帯域が広い場合は増やすと効果的
PORT 8080 サーバーのリスニングポート
AUTH_ENABLED false true にすると全リクエストで X-Auth-Key ヘッダーが必須になる
AUTH_KEY — AUTH_ENABLED=true の場合 必須  認証キー文字列

技術仕様

項目値説明
アップロード方式 マルチパートストリーミング ディスク書き出しなし。受信しながらS3互換ストレージへ転送
パートサイズ 32 MB S3マルチパートアップロードの分割単位（UPLOAD_PART_SIZE_MBで変更可）
並列パート数 8 1ファイルあたりの同時転送パート数（UPLOAD_CONCURRENCYで変更可）
同時リクエスト 最大 MAX_CONCURRENT_UPLOADS 超過リクエストはキューで待機
ストレージキー uploads/{元のファイル名} 同名ファイルは上書きされます
ReadHeaderTimeout 30 秒 スローロリス攻撃対策。ボディ転送にはタイムアウトなし
